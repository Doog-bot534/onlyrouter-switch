## 一、Switch 升级后的全貌

Switch 从"配置写入器"升级为**本地代理 + 桌面工作台**，三个面：

```
┌─ Switch.app (Electron) ───────────────────────────────┐
│  A. 配置面板（现有）   写 config.toml / settings.json   │
│  B. 生成面板（新，移植 playground）  文本/图像/视频 原生渲染 │
│  C. 本地代理（新，主进程常驻 HTTP）  拦截 Codex/CC 流量     │
└───────────────────────────────────────────────────────┘
                │ C 转发
                ▼
        api.onlyrouter.ai
```

价格/模型/约束**唯一事实源 = `onlyrouter.ai/api/models`**，定时拉取覆盖本地表。

## 二、本地代理（C）中间件链

Codex 等工具把 base_url 指向 `127.0.0.1:<port>/v1`（改 config.toml，配置注入式拦截），透传 Authorization。请求过这条链：

```
入: /v1/responses (Codex 发)
 → ① 协议翻译  Responses → Chat
 → ② 模态路由  文本? 图像/视频? → 图像/视频改道对应上游
 → ③ 智能路由  能力门槛 → 缓存经济性 → 迟滞，选 model
 → ④ 安全·出站 PII/密钥脱敏（可逆掩码）
 → 转发 api.onlyrouter.ai/v1/chat/completions
 ← ④ 安全·入站 投毒/注入检测（流式滑窗）
 ← ① 协议翻译  Chat → Responses（含 SSE 事件流）
 ← 还原占位符
出: Responses SSE (Codex 收)
```

生命周期照 LiteLLM 三类 hook：pre-call（改请求/脱敏）/ parallel-moderation（与转发并行，不加串行延迟）/ post-stream（逐 chunk 改写）。

**fail 策略分级**：PII 脱敏 **fail-close**（脱敏失败宁可不发）；智能路由/投毒检测 **fail-open**（组件挂了不堵死开发者）。

## 三、各层设计要点（已定决策）

### ① 协议翻译（P0，修实际 bug）
- 根因：`main.js:212` 写死 `wire_api="responses"`，DeepSeek 上游只有 chat → 404。`wire_api="chat"` 已被 Codex 废弃，必须翻译。
- **Codex 无状态已证实**（源码 client.rs 无 previous_response_id，store:false，每轮全量 input[] 重发）→ 无状态网关成立。蓝本 `MetaFARS/codex-relay` translate.rs/stream.rs。
- 实现：Node 主进程 HTTP server，Responses↔Chat 双向。**硬约束（验证所得，必须遵守）**：
  - 最小事件集 `response.created`(必带 response 对象{id,status}) → `output_item.added` → `output_text.delta*` → `output_item.done` → `response.completed`(**唯一终止、必带 usage**)。未知事件被忽略，reasoning 不发安全。
  - **assistant message item 必须带 `id`+`status`**(issue #12669)，否则下轮回放校验报错。
  - 请求映射：`instructions`→system；input[] 含 message/function_call/function_call_output/reasoning，**reasoning item 丢弃**；tools 扁平→嵌套 `{type:function,function:{...}}`；夹在 function_call/output 间的 system 消息挪到最前。
  - 流式工具调用：Chat tool_calls 按 index 累积、**流结束后一次性 emit**（`output_item.added`(function_call)→`function_call_arguments.delta`(完整args)→`output_item.done`），Codex 对 arguments 分片敏感。
  - 流未收到 [DONE] 不可存 assistant-with-tool_calls，要发 `response.failed`。
  - 接思维模型(reasoning_content)才需网关侧缓存回填；只接 DeepSeek chat 则纯无状态。
- config.toml 的 base_url 改指本地端口，wire_api 保持 responses；`read-codex-model` 的 configured 判定改看 `model_provider="onlyrouter"`（不再是 onlyrouter.ai 字样）。
- **两个入站适配器**：Codex `/v1/responses`（P0）+ Claude Code `/v1/messages`（后置，rdx stream.py 可复用），共用 ②③④ 中间件核心。

### ② 模态路由（P2）
- 读请求 + `model_type`（text/image/video/audio）判意图。**保守判定**：明确"生成/画一张…"才走图像，避免"读取 image 文件"误判。
- 图像/视频改道 OnlyRouter 对应端点，响应包装：Codex 终端只能回链接/markdown（渲染降级），**主场是生成面板 B**。
- 复用 playground 的 per-model 约束表（见第五节）和 grab.js 取视频逻辑。

### ③ 智能路由（P3）—— 决策顺序固定
1. **能力门槛（硬，先判）**：难/模糊/架构级 → 前沿模型，不比价。便宜模型做砸 = 重做，损失远超省的 token。
2. **简单任务才进缓存经济性比较**：
   - 留缓存模型：`前缀×cache_read_price + 新增×input_price + 输出×output_price`
   - 换便宜模型(冷)：`(前缀+新增)×便宜miss价 + 输出`
   - 反直觉结论：价差大时便宜模型连冷启动都赢 → 缓存惩罚**很少挡得住降级**；它真正的作用是**抑制反复横跳**（降了又切回前沿模型要按 miss 全价重建整段前缀缓存，那才是 5× 贵账单）。
3. **迟滞(hysteresis)**：一旦选定，本会话内粘住 model+provider（保 prompt cache 命中），除非任务难度越阈值才换；切换惩罚按剩余轮数摊销。
- 价格全锚 OnlyRouter 标价。

### ④ 安全网关
- **出站 PII（P1）**：A 档正则兜密钥(`AKIA`/`T3BlbkFJ`)+中国证件号(手机/身份证校验/Luhn)；**可逆占位符** `__OR_<类>_<hash8>__`（抄 rdx 设计，不用拟真假值）+ 按项目隔离映射表 + **加密落盘**（改进 rdx 的纯内存）；响应流式还原**抄 rdx stream.py 跨 chunk 缓冲算法但修掉 tool_use 不回写的 bug**。
  - **误脱敏防御（两家都没解决=差异化）**：allowlist + 已知非密模式豁免(UUID/SHA/MD5/base64图/RFC5737 IP/颜色码/AWS 示例 key) + 熵阈值 + 上下文关键词 + 置信度门槛；回归集直接用 rdx `tests/fixtures/tricky_false_positives.py`。
  - fail 策略分级：高置信密钥 fail-close（宁可不发）；软 PII 可配 fail-open/告警（避免 rdx 那种全 fail-open，也避免堵死开发者）。
  - 中文人名/地址需自训 ONNX 小模型（Node 无开箱，最大缺口）或 Presidio sidecar。
- **入站投毒（P1）**：L1 规则——Unicode 规范化剥隐藏字符 + 外泄链接(markdown 图片指外域/base64 query)检测 + 危险命令正则。中转站投毒是实证威胁(UCSB 实测在野有恶意路由器)，上游响应视为不可信。**两个竞品都没做 output 检测，这是我们领先点**。L2 ONNX 分类器(Prompt Guard 22M)作辅助信号。流式用 NeMo 式滑窗(256 token/overlap 64)。

## 四、生成面板（B）—— 移植 playground 进 Electron

playground 是单页 `index.html`(563 行，文本/图像/视频三 tab) + `api/grab.js`(视频 302 取流)。搬进 Electron 的改动与白捡好处：

| playground 现状 | 搬进 Switch |
|---|---|
| `vercel.json` rewrites 反代 /v1 解决 CORS | **删掉**——主进程/本地代理直接发，无 CORS |
| `api/grab.js` Vercel serverless 跟 302 | **搬进 main.js**——主进程 fetch `redirect:'manual'` 跟 302 拿签名直链(逻辑见 grab.js)，更简单 |
| 前端 `location.origin+'/v1'` | 走本地代理 C 的 `/v1`（顺带统一鉴权/日志） |
| index.html 全部 UI+逻辑 | 移植成 renderer 的一个 tab；图像 `<img>`/视频 `<video src=直链>` 原生渲染 |
| per-model size/约束动态填充 | 复用同一份约束表（第五节），与模态路由 ② 共享 |

## 五、Per-model 约束表（playground 已踩平，路由②与面板B共用）

| 模型 | 约束 |
|---|---|
| Seedream 4.5 | 强制最小 2K(2560×1440)，1K 报错 |
| Seedream 5.0/4.0 | 1K~4K |
| Qwen Image / Wan 2.7 | ~1MP（1328/1024 档） |

| Seedance Fast | 仅 480p/720p |
| Sora2 时长 | 4/8/12s；豆包 5/8/10s |

## 六、落地顺序

| 期 | 内容 | 验收 |
|---|---|---|
| **P0** | ① 协议翻译网关 + config 改写 | Codex 配 DeepSeek 不再 404，文本流式正常 |
| **P1** | ④ 出站脱敏 A 档 + 入站投毒 L1 + fetch-models 补字段 | 身份证/密钥到上游被掩码；外泄链接/危险命令拦截 |
| **P1** | B 生成面板移植（文本/图像/视频原生渲染 + grab 搬进 main） | App 内直接出图/出视频并渲染 |
| **P2** | ② 模态路由（Codex 发图改道，返回链接） | Codex 里"生成一张…"能出图链接 |
| **P3** | ③ 智能路由 A→B 档（能力门槛+缓存经济性+迟滞） | 简单任务降本可量化，会话内不横跳 |
| **P3+** | 中文 PII ONNX 模型、L2 投毒分类器、Presidio/LLM Guard sidecar | 按合规需求 |

> 技术选型注记：CC Switch 用 Rust/Tauri（更小体积/常驻内存）；本项目已 Electron，先不换，但若常驻开销敏感可评估。先读现成竞品 `aisecuritygateway`/`claude-code-redact` 架构再定差异化。
